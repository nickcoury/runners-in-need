# Browser Audit - 2026-03-25

## Scope

This audit covered:

- Local run at `http://localhost:4322` from this workspace on March 25, 2026
- Live browser verification against `https://runnersinneed.com` for read-only and intercepted-submit flows
- Playwright smoke coverage plus targeted manual Playwright interactions
- A small automated accessibility pass with `@axe-core/playwright`

## Environment Notes

No local `.env` file exists in this workspace. The app starts, but public routes still fail because auth/database configuration is pulled in before the request can degrade gracefully.

## Commands Run

```bash
npm run dev
npx playwright test tests/e2e/smoke.spec.ts --config=playwright.config.ts
npx playwright test tests/e2e/prod-smoke.spec.ts --config=playwright.prod.config.ts
```

I also ran targeted inline Playwright scripts against production to verify search, map toggles, need detail pages, pledge submission UX with an intercepted `/api/pledges` request, and mobile accessibility.

## Summary

- Local developer experience is currently broken without a real `.env`; even public pages returned `500`.
- The live site passed the existing production smoke suite.
- The most concrete live-site bug I reproduced is that search can hide every need card without showing the empty-state message.
- The main usability gaps remain around trust, discoverability, and post-pledge feedback rather than raw app stability.

## Findings

### 1. Critical - Local public pages fail with `500` when `.env` is absent

**Where:** local dev server, including `/`, `/about`, `/browse`, `/api/health`

**Evidence:**

- `npm run dev` booted on `http://localhost:4322`
- `npx playwright test tests/e2e/smoke.spec.ts --config=playwright.config.ts` failed 7/7
- Failures included:
  - `/` returned `500` instead of `200`
  - `/about` returned `500` instead of `200`
  - `/api/health` returned `500` instead of a degraded response
- Server logs showed `Error: TURSO_DATABASE_URL is not set`

**Likely cause:**

- [`src/lib/auth.ts`](/home/nick/claudeclaw/Repos/nickcoury/runners-in-need/src/lib/auth.ts) constructs the Drizzle adapter at module load via `getDb()`
- [`src/db/index.ts`](/home/nick/claudeclaw/Repos/nickcoury/runners-in-need/src/db/index.ts) throws immediately when `TURSO_DATABASE_URL` is missing
- That makes the app fail before public routes can fall back cleanly

**Impact:**

- New contributors cannot browse or smoke-test public pages locally without production secrets
- The current local setup is much more brittle than the README suggests

**Recommendation:**

- Make auth/database initialization lazy for public pages
- Ensure `/api/health` returns a clean degraded response even when DB env vars are missing
- Allow static/public routes to render without requiring a database connection

### 2. High - Search can hide all cards without showing the "No results" state

**Where:** live homepage search on `https://runnersinneed.com`

**Evidence:**

- I searched for `zzzzxyznonexistent12345`
- Visible need cards dropped to `0`
- `#no-results` stayed hidden

**Impact:**

- The user sees an apparently broken or blank listing area instead of clear feedback
- This is a real dead-end in the core browse journey

**Recommendation:**

- Fix the browse filtering logic so the `#no-results` state is shown whenever all `.need-card` elements are hidden by client-side filtering

### 3. Medium - Need cards do not link the organization name

**Where:** live homepage need cards

**Evidence:**

- First card org/location text rendered as `Refugee Runners Boston · Boston, MA`
- Org link count inside the card was `0`
- On the need detail page, an org link is present

**Impact:**

- Donors cannot jump straight from browse results to an organization profile
- This weakens trust-building and makes repeated giving harder

**Recommendation:**

- Make the organization line on need cards link to `/org/[id]`

### 4. Medium - Need detail pages still lack any share or copy-link action

**Where:** live need detail page

**Evidence:**

- Playwright found `0` share/copy buttons on the tested need detail page

**Impact:**

- Donors cannot easily forward a need to friends, teams, or clubs
- This slows organic sharing for the highest-intent page in the funnel

**Recommendation:**

- Add a small `Copy link` or `Share` control near the title or metadata row

### 5. Medium - Anonymous pledge success state does not help the donor track what happens next

**Where:** live need detail page after intercepted pledge submission

**Evidence:**

- I intercepted `/api/pledges` and submitted the form successfully
- `Pledge submitted!` appeared
- No visible prompt mentioned tracking, status follow-up, or creating an account

**Impact:**

- Anonymous donors get confirmation, but not continuity
- That leaves the donor uncertain about whether they will hear back and how

**Recommendation:**

- Expand the success state to say what happens next
- Add a lightweight prompt to create an account or explain how follow-up will happen by email

### 6. Medium - About page ends without a clear conversion path

**Where:** live `/about`

**Evidence:**

- `main a` count was `0` on the about page during the audit
- The page explains the service but does not provide an in-content next step

**Impact:**

- A convinced visitor has to rely on header/footer navigation instead of an explicit CTA
- That adds avoidable friction to a page meant to build trust

**Recommendation:**

- Add an in-content CTA block near the bottom:
  - `Browse Needs` for donors
  - `Become an Organizer` for programs

### 7. Medium - Mobile color contrast fails on the Accessories category badge

**Where:** live homepage, mobile viewport `375x812`

**Evidence:**

- `@axe-core/playwright` reported 1 violation on mobile home
- Violation: `color-contrast`
- Element:
  - `<span class="... bg-amber-600 text-white ...">Accessories</span>`
- Reported contrast ratio: `3.19:1`

**Impact:**

- Badge text does not meet WCAG AA contrast for normal text

**Recommendation:**

- Darken the badge background or switch to darker text for the Accessories badge

- Need detail pages loaded normally on production
